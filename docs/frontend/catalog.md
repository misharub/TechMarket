# Каталог TechMarket

## Маршруты

- `/catalog` - общий каталог. Показывает только крупные плитки корневых разделов.
- `/catalog/:categorySlug` для корневого раздела - показывает только его подкатегории и левый список основных разделов.
- `/catalog/:categorySlug` для конкретной подкатегории - показывает товарную выдачу, общие фильтры, сортировку и пагинацию.

Такой сценарий повторяет структуру крупных магазинов: сначала пользователь выбирает раздел, затем подкатегорию, и только после этого попадает в список товаров.

## URL-параметры выдачи

Фильтры появляются только на странице конкретной подкатегории.

- `q` - поисковая строка.
- `brand` - slug бренда.
- `priceFrom` - минимальная цена.
- `priceTo` - максимальная цена.
- `inStock=true` - только товары в наличии.
- `sort` - сортировка: `newest`, `priceAsc`, `priceDesc`, `titleAsc`.
- `page` - номер страницы.

При смене фильтра страница сбрасывается на первую. Пагинация сохраняет активные фильтры.

## Поведение интерфейса

- Общий каталог использует большие плитки с минималистичными смысловыми иконками категорий.
- Плитки категорий показывают только иконку и название, без счетчиков подкатегорий.
- Страница корневого раздела скрывает товары и фильтры, оставляя только подкатегории.
- Страница подкатегории показывает общий фильтр v1: поиск, бренд, цену, наличие и сортировку.
- Desktop: фильтры находятся слева, товары справа.
- Tablet/mobile: фильтры становятся компактным блоком над товарами.

## Данные

- Категории берутся из `GET /categories/tree`.
- Бренды берутся из `GET /brands`.
- Товары берутся из `GET /products` с параметрами из URL.
- `categorySlug` на backend включает выбранную категорию и дочерние категории.

## Состояния

- Loading: skeleton-плитки или skeleton-карточки.
- Error: сообщение о недоступности backend.
- Empty: сообщение "Ничего не найдено" и кнопка сброса фильтров.
- Кнопки "В корзину" и "Сравнить" пока визуальные, без мутаций.
