# Уровни категорий

В каталоге используется дерево категорий через поле `parentId`.

## Схема уровней

1. **1-й уровень** — корневые разделы header.  
   Пример: `Ноутбуки и компьютеры`.

2. **2-й уровень** — основные разделы внутри корневой категории.  
   Пример: `Аксессуары для ноутбуков`.

3. **3-й уровень** — конечные подразделы внутри раздела 2-го уровня.  
   Пример: `Рюкзаки для ноутбуков`.

```text
Ноутбуки и компьютеры
└── Аксессуары для ноутбуков
    └── Рюкзаки для ноутбуков
```

## Как добавлять

- Чтобы создать категорию **1-го уровня**, в поле `Родитель` выбрать `Без родителя`.
- Чтобы создать категорию **2-го уровня**, выбрать родителем категорию **1-го уровня**.
- Чтобы создать категорию **3-го уровня**, выбрать родителем категорию **2-го уровня**.

В форме создания категории в списке родителей показываются только категории 1-го и 2-го уровня. Это позволяет создавать дерево до 3 уровней и не превращать каталог в бесконечную вложенность.

## Правило для витрины

- В header показываются только категории **1-го уровня**.
- В боковом меню показываются категории **2-го уровня**.
- В правой части mega menu для выбранной категории 2-го уровня показываются её дети **3-го уровня**. Если у категории 2-го уровня нет детей, интерфейс не должен подставлять соседние разделы вместо них.
