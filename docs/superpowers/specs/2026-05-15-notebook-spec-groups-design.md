# Группы характеристик ноутбуков — дизайн

## Цель

Сделать для раздела `Ноутбуки` гибкую систему характеристик в стиле крупных магазинов: характеристики выводятся на карточке товара по смысловым группам, но у конкретных товаров может быть разное количество заполненных полей. Та же модель должна оставаться источником данных для подборок ноутбуков.

## Область первой реализации

Первая версия применяется только к разделу `Ноутбуки`.

- Другие разделы каталога не меняются.
- Текущая логика подборок ноутбуков сохраняется.
- Бренд остаётся отдельным условием подборки.

## Модель данных

### Группы характеристик

У раздела `Ноутбуки` появляются отдельные группы характеристик:

- `Классификация`
- `Экран`
- `Процессор`
- `Оперативная память`
- `Графический ускоритель`
- `Накопители данных`
- `Интернет и передача данных`
- `Питание`
- `Габариты и вес`

Группа хранит:

- название;
- порядок вывода;
- принадлежность к разделу категории.

### Шаблон характеристики

Каждая характеристика шаблона ноутбуков относится к одной группе и хранит:

- технический ключ;
- пользовательское название;
- тип значения;
- единицу измерения;
- обязательность;
- участие в сравнении;
- порядок внутри группы;
- признак, что характеристику можно использовать в подборках.

Часть полей обязательна, часть — необязательна. Если необязательная характеристика у товара не заполнена, на публичной странице она не показывается.

### Значения товара

Значения характеристик товара по-прежнему хранятся в `Product.specs`.

Это позволяет:

- не ломать текущую фильтрацию подборок;
- сохранить уже работающую валидацию по шаблону;
- не усложнять модель отдельной таблицей значений на этом этапе.

`additionalSpecs` остаются как запасной механизм для редких характеристик, которых ещё нет в шаблоне. Они не участвуют в подборках.

## Подборки ноутбуков

Подборки продолжают работать как сохранённые фильтры:

- по характеристике из шаблона: `характеристика -> значение`;
- либо по бренду как отдельному условию.

Примеры:

- `Игровые ноутбуки` → `Назначение = gaming`
- `Ноутбуки с Windows 11` → `Операционная система = Windows 11`
- `Ноутбуки с AMD Ryzen` → `Семейство процессора = AMD Ryzen`
- `Ноутбуки Apple MacBook` → `Бренд = apple`

В админке пользователь не должен вручную вводить технический ключ характеристики для подборки. Он выбирает характеристику из шаблона ноутбуков и затем значение.

## Админка раздела «Ноутбуки»

В форме редактирования раздела:

1. Добавляется управление группами характеристик.
2. Таблица характеристик показывает группу каждой характеристики.
3. При создании/редактировании характеристики выбираются:
   - группа;
   - название;
   - ключ;
   - тип;
   - единица;
   - порядок;
   - обязательность;
   - участие в сравнении;
   - можно ли использовать в подборках.
4. В блоке подборок:
   - для условия по характеристике выбирается характеристика из доступных для подборок;
   - значение выбирается из вариантов для `SELECT` или вводится вручную для других типов;
   - бренд остаётся отдельным типом условия.

## Админка товара

В форме товара раздела `Ноутбуки` характеристики показываются блоками по группам.

- Группы идут в порядке, заданном в админке раздела.
- Характеристики внутри группы идут по своему порядку.
- Админ заполняет только нужные значения.
- Пустые необязательные поля допустимы.

## Публичная страница товара

На странице ноутбука характеристики выводятся:

- только если значение заполнено;
- по группам;
- в порядке групп и характеристик;
- без пустых строк.

Это должно визуально приблизить блок характеристик к референсу со скриншота, сохранив более простую реализацию.

## Ошибки и ограничения

- Характеристика, участвующая в подборке, не должна удаляться, пока на неё завязана активная подборка.
- Для `SELECT`-характеристик значение подборки должно выбираться только из разрешённых вариантов.
- `additionalSpecs` не должны использоваться в подборках.
- Первая версия не переносит новую модель на другие разделы каталога автоматически.

## Не входит в первую версию

- Миграция всех категорий на новую модель.
- Полноценный универсальный конструктор сложных условий с `AND/OR`.
- Перенос всех значений характеристик из JSON в отдельные таблицы.
- Автоматическое наследование шаблонов между разными разделами.

## Критерий готовности

Для ноутбуков можно:

1. Создать группы характеристик и распределить поля по ним.
2. Заполнить ноутбук с разным количеством значений.
3. Увидеть на публичной карточке только заполненные значения, сгруппированные как в референсе.
4. Создать подборку, выбрав понятную характеристику из шаблона, а не вводя технический ключ вручную.
5. Сохранить текущие подборки `gaming`, `Windows 11`, `AMD Ryzen` и брендовые подборки без поломок.
